# 00. Product GDD

> 文档职责：说明《Bid or Bust》是什么、为什么成立、玩家体验是什么、项目当前选择的核心设计方向是什么。  
> 具体可实现规则见 `01_MVP_RULE_SPEC.md`。

---

## 1. High Concept

《Bid or Bust》是一款多人信息拍卖博弈游戏。玩家扮演旧货、打捞和非标资产市场中的竞拍者，在一个充满普通物品、二手资产和偶发神秘物的“平庸奇幻”世界里竞拍未知集装箱。

玩家通过以下信息判断箱子价值：

- 集装箱内物品的网格轮廓。
- 每轮公开的结构化情报。
- 自己每轮私下查看到的一件物品。
- 主动技能带来的一轮信息优势。
- 其他玩家的出价行为、Fold、All-in、是否使用技能等信号。

核心体验是：

> 我看到了一部分真相；别人也看到了一部分真相；每个人都在用报价伪装或暴露自己的判断。  
> 我必须决定：这个价格到底是机会、陷阱，还是我不敢错过的直接成交窗口。

## 2. Target Experience

### 2.1 玩家应该感受到什么

每个箱子都应该让玩家经历：

1. **好奇**：这个箱子里这些轮廓到底是什么？
2. **怀疑**：别人突然加价，是看到了好东西，还是在诈我？
3. **压迫**：如果我不跟进，别人会不会直接成交？
4. **承诺**：我现在报的价格不能降，判断错了就只能 Fold 或硬扛。
5. **高潮**：All-in、直接成交或第 5 轮最终成交。
6. **揭晓**：逐件开箱，确认自己到底是捡漏还是接盘。
7. **延续**：这一箱赚亏会影响本场后续箱子和局外仓库。

### 2.2 不是要做什么

当前项目不是：

- 纯开盲盒游戏。
- 纯模拟经营游戏。
- 纯仓库拍卖写实模拟。
- 纯扑克或德州扑克变体。
- 纯数学估值游戏。
- 以道具卡为主的卡牌游戏。

它是一个以“信息 + 出价承诺 + 开箱反馈”为核心的短局博弈游戏。

## 3. Experience Pillars

### 3.1 信息差，而不是完全随机

箱子里具体是什么仍然有赌博性，但玩家不会完全盲拍。玩家可以通过轮廓、品质、分类、私查、技能和公共信息逐渐形成估值。

设计重点：

- 信息必须足够结构化，能让玩家推理。
- 信息不能过强，否则变成明牌。
- 每个玩家看到的信息不同，才能产生心理博弈。

### 3.2 报价必须代表承诺

如果玩家能随意报高价再降价，前几轮报价就会变成无成本噪音。因此当前设计采用“承诺价”：

- 只能保持或提高。
- 不能降低。
- Fold 后退出当前箱子。
- All-in 若最高则立即成交。

这让诈唬依然可行，但诈唬必须承担风险。

### 3.3 每轮都可能结束

直接成交机制让前 4 轮不会沦为试探垃圾时间。

玩家每轮都必须考虑：

- 我是否要压高价直接拿走？
- 如果我不提高承诺价，别人是否会直接成交？
- 我是否应该花钱维持第二名压力，避免领先者触发成交？

### 3.4 快节奏结算爽感

每个箱子结束后立刻开箱并场内变现。

这是为了：

- 保证每个箱子有独立情绪高潮。
- 让多箱场次可以继续推进。
- 让 All-in 不是“赌完后坐牢”。
- 让玩家立刻知道自己判断是否正确。

### 3.5 外循环不是只攒钱

长期目标不是单纯赚更多钱，而是：

- 获得现金。
- 保留关键藏品。
- 用藏品证明自己有资格进入更高级别拍卖场。
- 在更高 stake 的场里获得更稀有的藏品。

## 4. Core Loops

### 4.1 Moment-to-Moment Loop

```text
看轮廓 / 看信息
→ 判断可能价值
→ 观察报价信号
→ Hold / Raise / Fold / All-in
→ 等待公开结果
→ 更新判断
```

### 4.2 Container Loop

```text
展示箱子轮廓与场次参数
→ 最多 5 轮信息报价
→ 直接成交或第 5 轮成交
→ 赢家开箱
→ 总价值公开
→ 场内变现
→ 进入下一个箱子
```

### 4.3 Session Loop

```text
选择拍卖场
→ 支付入场费
→ 划转托管资金
→ 连续拍多个箱子
→ 场内资金变化
→ 场结束结算
→ 仓库处理藏品
```

### 4.4 Meta Loop

```text
参加低级场
→ 赚钱和获得藏品
→ 保留准入藏品
→ 解锁更高级场
→ 进入更高 stake 的拍卖
→ 获得更稀有资产
```

## 5. World and Tone

### 5.1 世界观：平庸奇幻旧货经济

这个世界里，神秘物品不是史诗神器，也不是毁灭世界的禁忌，而是旧货市场、打捞渠道、破产清算和异常品登记系统中的非标资产。

普通物品占多数：

- 旧收音机
- 船钟
- 工具箱
- 破损家具
- 工业零件
- 绝版杂志
- 拍卖槌
- 陶瓷摆件

神秘物偶尔出现：

- 永远有电但没有接口的电池。
- 会指向昨天的罗盘。
- 自己播报天气的收音机。
- 未登记的低语珍珠。
- 不该出现在普通清仓场里的异常封蜡。

### 5.2 气质关键词

- 日常
- 旧货
- 拍卖综艺感
- 市侩
- 轻松
- 荒诞
- 非标资产
- 港口 / 仓库 / 清仓场 / 典当行
- 神秘物调查很随便

### 5.3 避免的方向

避免变成：

- 过重克苏鲁恐怖。
- 纯魔法史诗。
- 赛博朋克战斗游戏。
- 严肃 SCP 管理模拟。
- 只有搞笑没有风险的无厘头。

## 6. Positioning Against BidKing

BidKing 已经验证了以下机制的吸引力：

- 盲标 / 暗标。
- 直接成交阈值。
- 快节奏开箱。
- 玩家关注其他人的报价。
- 角色技能带来的心理战。

《Bid or Bust》不应简单复制，而应差异化：

| 维度 | BidKing 倾向 | Bid or Bust 方向 |
| --- | --- | --- |
| 拍卖对象 | 更偏单藏品 / 单箱快速刺激 | 多物品集装箱，至少 10 件物品 |
| 信息 | 报价公开更强，节奏很快 | 结构化信息 + 私查 + 技能一轮优势 |
| 报价 | 可自由调价带来无成本试探风险 | 承诺价只能升不能降 |
| 成交压力 | 直接成交很强 | 保留并强化，与 All-in 结合 |
| 道具 | 存在感低，略啰嗦 | MVP 移除；未来与特殊藏品融合 |
| 外循环 | 收藏展示 / 网游经济倾向 | 现金 + 藏品资质 + 拍卖场准入 |
| 世界观 | 现代拍卖 / 藏品 | 平庸奇幻、旧货、非标资产 |

核心差异总结：

> BidKing 更像“快速拍卖心理战”。  
> Bid or Bust 应该是“快速拍卖心理战 + 网格轮廓推理 + 藏品资质外循环”。

## 7. Design Principles

### 7.1 先让规则产生戏剧性，再加内容

技能、物品、角色都应该服务于核心拍卖张力。不要先堆大量道具、收藏或剧情。

### 7.2 所有有效信息都要结构化

如果信息会影响估值或报价，就必须能被程序理解。

好的信息：

```text
显示随机 2 件物品的品质。
显示箱中是否存在 Oddity。
显示最大轮廓物品的分类。
```

不好的有效信息：

```text
这个箱子不太对劲。
有人觉得这里有好东西。
```

氛围文本可以存在，但底层必须对应结构化事实。

### 7.3 UI 帮玩家算下限，不替玩家出价

系统可以显示已知保底价值，但不显示推荐出价。

原因：

- 保底是信息整理。
- 推荐价会替玩家玩游戏。

### 7.4 复杂度要放在推理，不放在输入

玩家应该思考：

- 这个轮廓可能是什么？
- 他为什么加价？
- 我是否相信自己的信息？

玩家不应该思考：

- 我该输入 43127 还是 43128？

因此 Raise 用固定 step，不用任意数字输入。

### 7.5 外循环服务内循环

仓库、藏品、商店、拍卖场解锁都应该反过来增强玩家对集装箱竞拍的期待，而不是把游戏变成独立经营系统。

## 8. Current Product Direction

当前建议的普通模式：

- 一场拍卖包含 3 个集装箱。
- 每个集装箱最多 5 轮报价。
- 每箱至少 10 件物品。
- 普通模式公开玩家资金、最高价、成交价、箱子总价值和净收益。
- 不公开完整报价榜和赢家内容物明细。
- 每轮有公共信息。
- 每名玩家每轮可私查 1 件物品。
- 玩家可主动使用随机动态技能，信息下一轮公开。
- 承诺价只能升不能降。
- Raise 使用固定 step。
- All-in 若成为最高价则立即成交。
- 非 All-in 情况使用递减直接成交阈值。
- 成交后场内变现。
- 特殊藏品可保留，用于局外准入。
- 道具系统 Demo 暂不做。

## 9. MVP Success Criteria

MVP 不是要验证美术、剧情或商业化，而是验证：

1. 玩家是否能理解 5 轮信息报价。
2. 玩家是否会因为直接成交压力而每轮认真出价。
3. 玩家是否会根据私查和公共信息形成不同判断。
4. 玩家是否会因为别人使用技能或加价而产生心理猜疑。
5. All-in 是否带来足够高潮，而不是破坏局面。
6. 场内变现是否让多箱局更有节奏。
7. 藏品保留是否形成有意义的局外选择。
8. 玩家是否愿意马上再开一场。

## 10. Future Product Directions

可能扩展方向：

- 高级场使用资金模糊规则。
- 特殊藏品可消耗为局内效果。
- 商店 / 典当 / 抵押。
- 更多角色被动。
- 更多 Oddity 风险。
- 单人 AI。
- Godot 客户端。
- 权威服务器。
- Steam 版本。
