# Bid or Bust Design Docs

> 本目录是《Bid or Bust》的设计文档入口。  
> 它把早期长篇 GDD 和多轮讨论拆成多个职责清晰的 Markdown 文件，避免单个文档越来越臃肿，也确保有价值的设计取舍不会在拆分中丢失。

## 文档拆分原则

本目录按游戏设计最佳实践拆分：

1. **产品级 GDD**：解释游戏是什么、为什么成立、体验支柱是什么。
2. **规则规格书**：把可实现的 MVP 规则写成工程可读的状态机与判定逻辑。
3. **系统设计文档**：分别承载竞价、信息、经济、外循环、内容等核心系统。
4. **内容指南**：约束世界观、物品、角色和文本风格，避免内容扩展时跑偏。
5. **决策日志**：保留讨论中有价值的方案、取舍、被放弃设计和未来可能回收的想法。
6. **MVP 范围与路线图**：把“现在必须做”和“以后再做”分开。

## 文档地图

### 00_PRODUCT_GDD.md

高层游戏设计文档。说明游戏概念、目标受众、体验支柱、核心循环、世界观方向、与 BidKing 的差异化，以及当前版本的总体设计结论。

适合用于：

- 对外解释项目。
- 新成员快速了解游戏。
- 判断新设计是否符合项目方向。

### 01_MVP_RULE_SPEC.md

当前 MVP 的权威规则规格。描述一场拍卖、一个集装箱、每轮报价、承诺价、直接成交、All-in、Fold、场内变现、公开信息等具体规则。

适合用于：

- 开发 core 状态机。
- 写测试用例。
- 和 UI / 网络层对齐交互流程。

### 02_INFORMATION_AND_BIDDING.md

深入说明信息系统与竞价系统如何互相咬合。包括公共信息、私人查看、主动技能、下一轮公开、已知保底价值、报价公开策略、直接成交阈值等。

适合用于：

- 设计技能。
- 设计公共情报。
- 调整竞价压力。
- 分析玩家是否会搭便车或无成本诈唬。

### 03_ECONOMY_META_AND_SESSION.md

经济、场次和外循环设计。包括入场费、托管资金、公开资金、箱子参考估值范围、场内变现、多箱场次、仓库、藏品资质、高级拍卖场准入等。

适合用于：

- 设计拍卖场等级。
- 设计外循环。
- 平衡资金压力。
- 处理藏品保留与出售。

### 04_CONTENT_AND_WORLD_GUIDE.md

内容与世界观指南。定义“平庸奇幻”的世界观基调，物品分类、品质、轮廓、Oddity、Contraband、角色、技能和文本风格。

适合用于：

- 扩充物品池。
- 写角色和技能。
- 统一 UI 文案和物品描述。

### 05_DECISION_LOG.md

设计决策记录。保留关键讨论、被比较的方案、选择原因、暂缓原因和未来可回收方向。

适合用于：

- 追溯为什么不照搬 BidKing。
- 理解为什么改成承诺价、场内变现、资金公开。
- 防止旧问题被重复讨论。

### 06_MVP_SCOPE_AND_ROADMAP.md

MVP 范围、测试目标和后续阶段计划。把必须实现、可以暂缓、未来扩展、高级场规则、Godot / Steam 迁移方向放在一起。

适合用于：

- 排期。
- GitHub issues 拆分。
- 判断某个功能是否该进 Demo。

## 权威性约定

- 规则实现以 `01_MVP_RULE_SPEC.md` 为准。
- 设计原因和取舍以 `05_DECISION_LOG.md` 为准。
- 经济和外循环以 `03_ECONOMY_META_AND_SESSION.md` 为准。
- 内容扩展以 `04_CONTENT_AND_WORLD_GUIDE.md` 为准。
- 如果文档之间冲突，优先更新被冲突文档，并在 `05_DECISION_LOG.md` 留下变更记录。

## 当前核心结论摘要

当前普通模式采用：

- 一场拍卖包含多个集装箱，MVP 推荐 3 个。
- 每个集装箱最多 5 轮报价。
- 每轮有公共信息、私人查看和可选主动技能。
- 技能信息只有一轮优势，下一轮公开。
- 玩家使用承诺价，承诺价只能保持或提高，不能降低。
- Raise 使用固定 step，不使用任意数字输入。
- Fold 后退出当前箱子，不能重新加入。
- All-in 使用全部当前托管资金；若成为最高价，立即成交。
- 非 All-in 情况下，第 1～4 轮按直接成交阈值提前成交，第 5 轮最高价自然成交。
- 普通模式公开当前最高价、资金、成交价、箱子总价值和净收益。
- 不公开每个人的具体报价，不公开最高价玩家身份，不公开赢家内容物明细。
- 箱子成交后场内变现，赢家看到完整内容物，其他玩家只看到总价值和净收益。
- 道具系统 MVP 暂不做；未来与特殊藏品融合。
