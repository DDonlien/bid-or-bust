# Roadmap

## Phase 0: Core Prototype

- Vite + TypeScript H5 Demo。
- 本地热座可玩。
- 至少 4 名玩家、5 轮报价、10 件以上物品。
- 每轮免费查看 1 件物品。
- 随机动态技能和下一轮公开。
- PeerJS Host-as-Server 壳。

## Phase 1: Playable Itch Demo

- 完成房间创建、加入、重连的最低可用体验。
- 加入 4-8 个角色，给每个角色明确的被动倾向和 4 个动态技能。
- 做一版更清晰的图鉴候选 UI。
- 增加 20-40 个物品、多个风险等级和来源标签。
- 补充拍下后的极简售卖界面。
- 加入声音、粒子、数字跳动反馈。

## Phase 2: Retention Test

- 调整经济系统，减少赢家滚雪球。
- 做落后补偿或领先者压力机制。
- 增加短局模式、目标金额模式、淘汰模式。
- 记录匿名对局数据：平均成交价、弃牌轮次、技能使用率、利润分布。

## Phase 3: Commercial Track

- Godot 客户端。
- Node.js 或 Go 权威服务器。
- Steam 平台能力。
- 正式匹配、房间、观战、断线重连。
- 反作弊私有情报裁剪。

## Open Design Questions

- 单局胜利条件：固定箱数结算，还是目标金额结算？
- 入场费按单场收，还是每个箱子收？
- Follow 是否需要临时冻结当前出价金额？
- 第 5 轮是否改为暗标，还是保持公开 Raise/Fold？
- 神秘物品只影响估值，还是影响同箱其他物品？
